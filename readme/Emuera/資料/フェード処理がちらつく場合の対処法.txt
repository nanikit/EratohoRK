1819로 묘화 처리를 변경했습니다
이 변경에 의해, 페이드와 같은 화면 효과의 재생중에 화면이 아른거리는 경우가 있습니다.
이것은 지금까지 표면화하지 않았던 ERB 스크립트의 버그의 표면화입니다.
이 문제는 1818 이전의 Emuera에서도 「프레임 매초」를 매우 큰 값으로 하는 것으로 재현 할 수 있습니다.

（1）최종 사용자님에게
Emuera의 콘피그로 「표시」탭의 「프레임 매초」를 1으로 설정해 주세요
아마 아른거리지 않게 됩니다.


（2）ERB 개발자님에게
화면의 아른거림은 CLEARLINE를 실시한 순간 상태가 묘화 되어 버리는 것이 원인입니다.
페이드와 같은, CLEARLINE를 사용한 화면의 개서를 빈번하게 실시하는 처리를 했을 경우, REDRAW 명령을 이용해 주세요.
	REDRAW 0
	CLEARLINE x
	PRINTL hoge
	REDRAW 1
	TWAIT 100, 1
REDRAW 0에 의해 묘화 처리가 억제되어 다음에 REDRAW 1을 실시할까 유저의 입력 대기 상태가 될 때까지 화면 묘화를 하지 않게 됩니다
결과적으로 CLEARLINE 후, PRINT전에 묘화를 하지 않게 되어, 화면의 아른거림을 막을 수가 있습니다.

덧붙여 1819에서도 이 문제가 발생하지 않는 경우여도, 잠재적인 버그인 것은 바뀌지 않습니다.
개발 환경에 있어 화면이 아른거리지 않았다고 해도 유저의 환경이나 「프레임 매초」의 설정하기 나름으로 아른거릴 가능성이 있습니다.
REDRAW 명령을 활용해 주세요.
